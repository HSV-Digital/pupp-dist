# Configuration

The root [.env.example](../.env.example) file is the source of truth for runtime configuration.

## App-local examples

- [apps/web/.env.example](../apps/web/.env.example) is a derived subset for running the Next.js app directly.
- [apps/api/.env.example](../apps/api/.env.example) is a derived subset for running the NestJS API directly.
- Keep the root [.env.example](../.env.example) as the canonical partner-deployment contract. The app-level files are convenience examples, not separate sources of truth.

## Runtime groups

- App URLs and routing
- Authentication
- Database
- Redis / queueing
- Blob storage and CDN
- Analytics
- PDF and link-signing secrets
- Demo gating and partner-specific controls

## Canonical variables

- `NEXT_PUBLIC_APP_URL`: public web origin used by the Next.js app and metadata.
- `NEXT_PUBLIC_API_BASE_URL`: public API origin used by browser requests.
- `API_BASE_URL`: server-side API origin used by the web app in Docker or private-network deployments.
- `FRONTEND_URL`: public web origin used by the API for CORS and generated links.
- `API_PUBLIC_BASE_URL`: public API origin embedded in API-generated links.
- `PARTNER_UPLOAD_URL`: public reseller page linked from partner-ready assets.
- `NEXT_PUBLIC_ASSET_BASE_URL`: optional asset or CDN origin for bundled public files.
- `DATABASE_URL`: canonical PostgreSQL connection string.
- `REDIS_URL`: canonical Redis connection string.
- `AUTH_SECRET`: primary auth secret. `NEXTAUTH_SECRET` is kept as a compatibility alias.
- `NEXT_PUBLIC_ENABLE_DEMO`: demo kill switch for both web and API demo-only routes.
- `AZURE_STORAGE_ACCOUNT_NAME`: enables Azure Blob-backed asset storage when set.
- `POSTHOG_PROJECT_TOKEN`: enables server-side PostHog event capture when set.
- `NEXT_PUBLIC_POSTHOG_KEY`: enables client-side PostHog capture when set.

## Practical grouping

### App URLs

These must stay consistent with your deployed origins:

- `NEXT_PUBLIC_APP_URL`
- `NEXT_PUBLIC_API_BASE_URL`
- `API_BASE_URL`
- `FRONTEND_URL`
- `API_PUBLIC_BASE_URL`
- `PARTNER_UPLOAD_URL`

See [Setup Guide](./SETUP.md) and [Deployment](./DEPLOYMENT.md).

### Authentication

These drive the internal and reseller sign-in flows:

- `AUTH_SECRET`
- `NEXTAUTH_SECRET`
- `AZURE_AD_CLIENT_ID`
- `AZURE_AD_CLIENT_SECRET`
- `AZURE_AD_RESELLER_CLIENT_ID`
- `AZURE_AD_RESELLER_CLIENT_SECRET`
- `GOOGLE_CLIENT_ID`
- `GOOGLE_CLIENT_SECRET`
- optional tenant controls such as `ALLOWED_TENANT_IDS`

See [Authentication](./services/authentication.md).

### Database and Redis

Primary canonical values:

- `DATABASE_URL`
- `REDIS_URL`

Compatibility fallbacks remain supported for the API:

- `PGHOST`, `PGPORT`, `PGUSER`, `PGPASSWORD`, `PGDATABASE`
- `REDIS_HOST`, `REDIS_PORT`, `REDIS_PASSWORD`, `REDIS_DB`

See [Database](./services/database.md) and [Redis](./services/redis.md).

### Storage

Blob-backed generation and upload flows use:

- `AZURE_STORAGE_ACCOUNT_NAME`
- `AZURE_STORAGE_ACCOUNT_KEY`
- `AZURE_STORAGE_CONTAINER_NAME`
- `AZURE_CDN_BASE_URL`
- `BLOB_SAS_EXPIRY_SECONDS`

See [Storage](./services/storage.md).

### Analytics

Client analytics:

- `NEXT_PUBLIC_POSTHOG_KEY`
- `NEXT_PUBLIC_POSTHOG_HOST`

Server analytics and admin analytics:

- `POSTHOG_PROJECT_TOKEN`
- `POSTHOG_CAPTURE_HOST`
- `POSTHOG_PERSONAL_API_KEY`
- `POSTHOG_ENDPOINT_API_KEY`
- `POSTHOG_WEB_PROJECT_ID`
- `POSTHOG_QUERY_HOST`

See [Analytics](./services/analytics.md).

### Generated asset and token settings

These are usually generated by the setup wizard and should be kept secret:

- `DL_TOKEN_ENCRYPTION_KEY`
- `PDF_DL_TOKEN_SECRET`
- `PDF_PASSWORD_ENCRYPTION_KEY`
- `PPT_TOKEN_SECRET`
- `RESELLER_API_TOKEN_SECRET`

Worker and generation tuning:

- `PDF_RENDER_TIMEOUT_MS`
- `PDF_MAX_CONCURRENCY`
- `PDF_CACHE_TTL_SECONDS`
- `PDF_ASYNC_PART_SIZE`
- `QPDF_BINARY`

## Notes

- `NEXT_PUBLIC_*` variables are exposed to the browser and must not contain secrets.
- Keep production secrets in your deployment platform's secret manager where possible.
- The setup wizard writes a root `.env` file for local and simple self-hosted deployments.
- Legacy aliases remain supported during transition, including `NEXT_PUBLIC_API_URL`, `NEXT_PUBLIC_CDN_URL`, `SITE_URL`, `NEXT_PUBLIC_SITE_URL`, `APP_ORIGIN`, and `NEXT_PUBLIC_APP_ORIGIN`, but they should not be used as the primary documented interface.

Service-specific setup guidance lives under [docs/services](./services).
